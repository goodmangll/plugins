# AI 主动提问工作流程

## 工作流程概述

AI 主动提问按照以下五个阶段进行引导：

1. 问题分析与意图识别
2. 思维模型选择
3. 提问会话执行
4. 深度追问（可选）
5. 总结与反馈

> **注意**：此文档是工作流程的参考文档，实际执行由 `thinking-assistant` Skill 作为核心协调器完成。

---

## 第一阶段：问题分析与意图识别

### 目标

深入分析用户提出的问题或场景，理解用户的真实意图和需求。

### 执行过程

1. **理解用户输入**：
   - 用户提出了什么问题？
   - 用户的真实意图是什么？（学习、决策、创新、反思等）
   - 用户可能处于什么知识水平？

2. **分析问题类型**：
   - 是需要深入理解某个概念吗？
   - 是需要做决策选择吗？
   - 是需要产生创意吗？
   - 是需要探索价值观吗？

3. **识别关键词**：
   - "理解"、"学习"、"解释" → 深度学习需求
   - "为什么"、"应该"、"选择" → 决策分析需求
   - "创意"、"想法"、"新角度" → 创新需求
   - "价值观"、"观点"、"看法" → 自我反思需求

4. **必要时提出澄清问题**：
   - 如果问题不够清晰，可以提出澄清问题
   - 确保对问题的理解准确

### 输出格式

```
## 问题分析

**用户输入**：[用户原始问题]
**识别意图**：[学习/决策/创新/反思]
**问题类型**：[具体类型描述]
**知识水平评估**：[初级/中级/高级]
**核心关注点**：[用户最关心的是什么]
```

---

## 第二阶段：思维模型选择

### 目标

根据问题分析结果，自动选择最合适的思维模型。

### 执行过程

1. **匹配思维模型**：
   - 根据 `thinking-models.md` 中的选择指南
   - 匹配最合适的思维模型
   - 考虑是否需要模型组合

2. **说明选择理由**：
   - 解释为什么选择这个模型
   - 说明该模型如何帮助用户解决问题

3. **确认或调整**（可选）：
   - 可以向用户确认是否使用这个模型
   - 如果用户有偏好，可以调整模型选择

### 输出格式

```
## 思维模型选择

**选择模型**：[费曼学习法/苏格拉底诘问法/第一性原理/水平思考法]
**选择理由**：
- [理由1]
- [理由2]

**应用场景**：[该模型适用场景的简要说明]
```

---

## 第三阶段：提问会话执行

### 目标

执行选定的思维模型，通过提问引导用户深度思考。

### 输出优先原则（关键）

**必须在调用 AskUserQuestion 工具前，完成以下文本输出**：

1. 完成第一阶段的"问题分析"输出
2. 完成第二阶段的"思维模型选择"输出

只有输出完成后，才能开始调用 AskUserQuestion 进行提问。

### 执行过程

根据选定的思维模型，按照 `thinking-models.md` 中对应的**AI 实施策略**进行提问。

> 各模型的具体实施策略详见 `thinking-models.md`：
> - 费曼学习法：[执行流程](../thinking-models.md#费曼学习法-ai-实施策略)
> - 苏格拉底诘问法：[执行流程](../thinking-models.md#苏格拉底诘问法-ai-实施策略)
> - 第一性原理：[执行流程](../thinking-models.md#第一性原理-ai-实施策略)
> - 水平思考法：[执行流程](../thinking-models.md#水平思考法-ai-实施策略)

## 第四阶段：深度追问（可选）

### 目标

在基础提问后，根据用户的反应进行更深入的追问。

### 判断是否需要深度追问

以下情况可以进行深度追问：

1. 用户回答简短或表面化
2. 用户表现出困惑或犹豫
3. 问题特别复杂，需要更深入探讨
4. 用户主动要求更多讨论

### 深度追问策略

1. **追问原因**：
   - "为什么你这么认为？"
   - "这个结论是怎么得出的？"

2. **追问假设**：
   - "如果去掉这个假设，结论会变吗？"

3. **追问影响**：
   - "这样做的影响是什么？"
   - "长期来看会怎么样？"

4. **追问替代**：
   - "有没有其他方式？"
   - "如果情况变了呢？"

---

## 第五阶段：总结与反馈

### 目标

总结整个提问过程，提供有价值的反馈。

### 执行过程

1. **回顾讨论过程**：
   - 简要回顾主要问题和回答
   - 突出关键洞察

2. **总结发现**：
   - 用户发现了什么新见解？
   - 用户的知识盲区是什么？
   - 有哪些重要的假设被揭示？

3. **提供反馈**：
   - 肯定用户的思考
   - 指出可以进一步探索的方向
   - 推荐相关的学习资源或方法

4. **建议后续行动**：
   - 建议下一步可以做什么
   - 推荐相关的思维模型或方法
   - 提供实际应用的建议

### 输出格式

```
## 总结与反馈

### 讨论回顾
[简要回顾主要问题]

### 关键洞察
- [洞察发现1]
- [洞察发现2]

### 发现的知识盲区
- [盲区1]
- [盲区2]

### 进一步探索建议
- [建议1]
- [建议2]

### 后续行动
[行动步骤说明]
```

## 协调器职责

### 保持引导者角色

- 不直接给出答案，而是引导用户自己思考
- 提问是开放的，避免封闭式问题
- 给予用户足够的思考和表达空间

### 确保思维质量

- 追问要有逻辑，逐步深入
- 避免跳跃式的提问
- 确保每个问题都有明确目的

### 提供积极反馈

- 对用户的思考给予肯定
- 鼓励用户表达自己的观点
- 营造安全和开放的讨论氛围

### 灵活调整策略

- 根据用户的反应调整提问方向
- 如果用户困惑，可以简化或换个角度
- 如果用户表现出兴趣，可以深入探讨

---

## 退出条件

以下情况下可以结束提问会话：

1. 用户明确表示结束
2. 用户表示已经理解或得到答案
3. 用户表现出满意或感谢
4. 已经进行了充分的讨论和探索
5. 达到预设的问题数量或轮次

在结束前，确保完成总结与反馈阶段。

---

## 实施注意事项

### 通用原则

1. **循序渐进**：不要一次性问太多问题，逐步深入
2. **耐心引导**：给用户足够的思考时间
3. **积极反馈**：对用户的回答给予肯定和鼓励
4. **灵活调整**：根据用户的反应调整提问策略
5. **总结反馈**：定期总结讨论进展

### 避免误区

1. **不要直接给出答案**：保持引导者角色
2. **不要问封闭式问题**：避免"是/否"问题
3. **不要打断用户**：让用户完整表达
4. **不要催促**：给用户充分思考时间
5. **不要评判**：避免对用户观点进行评判
