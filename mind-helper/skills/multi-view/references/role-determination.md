# 动态角色决定机制

## 核心问题

协调器需要回答：

> **"这个世界上什么人适合探讨这个议题？"**

> 详细的角色决定流程请参见 `role-determination-process.md`

---

## 角色类型说明

### 具体人物角色

**优点**：
- 有独特个性、说话风格、已知行为模式
- 更容易产生生动的讨论
- 适合需要强烈个人观点的场景

**示例**：
- 史蒂夫·乔布斯 - 产品设计师、完美主义者
- 埃隆·马斯克 - 愿景家、大胆创新者
- 蒂姆·库克 - 运营专家、务实管理者
- 杰夫·贝佐斯 - 长期主义者、客户至上
- 彼得·蒂尔 - 逆向思维者、颠覆创新者
- 玛丽·米克尔 - 互联网分析专家

### 抽象职位角色（必须添加详细画像）

**优点**：
- 代表专业领域
- 适合技术问题
- 更具普遍性

**强制要求**：
- 必须包含**性格特征**描述：内向/外向、谨慎/大胆、理性/感性等
- 必须包含**说话风格**描述：直接/委婉、技术化/通俗化、幽默/严肃等
- 必须包含**价值观**描述：注重安全/效率、追求完美/实用、长期主义/短期主义等
- 必须包含**决策偏好**描述：数据驱动/直觉判断、保守/创新等

**反例（禁止）**：
- ❌ "产品经理" - 过于空泛，没有任何人物特征
- ❌ "开发者" - 没有具体的性格或风格
- ❌ "设计师" - 缺少价值观和决策偏好

**正例（推荐）**：
- ✅ "具有乔布斯式完美主义的产品设计师" - 有明确性格特征
- ✅ "注重用户隐私的安全工程师" - 有明确价值观
- ✅ "追求极致性能的后端架构师" - 有明确决策偏好
- ✅ "偏好渐进式增强的前端开发者" - 有明确技术哲学

## 圆桌会议角色决定示例

### 具体人物角色示例

**议题：如何设计一个颠覆性的 AI 产品？**

可能的角色：

| 角色 | 身份描述 | 专业视角 | 说话风格 |
|------|----------|----------|----------|
| 史蒂夫·乔布斯 | 已故苹果联合创始人，产品设计大师 | 用户体验至上、极简主义、拒绝妥协 | 直率、激情、善用比喻、追求完美 |
| 埃隆·马斯克 | 特斯拉/SpaceX CEO，愿景家 | 第一性原理思维、颠覆性创新、解决人类根本问题 | 直接、技术导向、未来主义、强调风险和回报 |
| 蒂姆·库克 | 苹果 CEO，运营大师 | 供应链优化、效率提升、可持续发展 | 冷静、数据驱动、务实、强调执行力 |
| 萨姆·奥特曼 | OpenAI CEO，AGI 追求者 | AI 安全、通用人工智能、技术民主化 | 平衡、谨慎但乐观、强调责任 |
| 乔治·索罗斯 | 投资大鳄，哲学思考者 | 市场机制、反身性理论、开放社会 | 深思熟虑、理论化、强调不确定性 |

## 角色描述规范

### 具体人物角色描述模板

```
角色姓名：[真实人物姓名]

身份描述：
- 职业/身份
- 主要成就或领域
- 代表性作品或项目

专业视角/创意视角：
- 核心思维模式
- 关注的重点领域
- 独特的观点或方法

说话风格：
- 语言特点（直接/委婉、技术化/通俗化等）
- 表达方式（比喻/故事/数据等）
- 情绪特征（冷静/激情/幽默等）
```

### 抽象职位角色描述模板

```
角色名称：[具体化的职位名称，必须包含人物特征]

身份描述：
- 专业背景和经验
- 专注领域
- 代表性能力或特长

专业视角/创意视角：
- 核心分析角度
- 关注的重点维度
- 独特的见解或方法论

性格特征：
- 内向/外向
- 谨慎/大胆
- 理性/感性
- 细节/宏观
- 其他相关性格特征

说话风格：
- 语言特点（直接/委婉、技术化/通俗化等）
- 表达方式（图表化/故事化/数据化等）
- 情绪特征（冷静/激情/幽默/严肃等）

价值观：
- 注重安全/效率
- 追求完美/实用
- 长期主义/短期主义
- 用户至上/技术优先
- 其他核心价值观

决策偏好：
- 数据驱动/直觉判断
- 保守/创新
- 渐进/激进
- 风险规避/风险承担
- 其他决策倾向
```

### 角色描述检查表

在确定角色后，检查以下项目：

- [ ] 角色名称不是空泛的职位名称（如"产品经理"、"开发者"）
- [ ] 具体人物角色：有真实的姓名和已知的个性特征
- [ ] 抽象职位角色：包含详细的性格特征描述
- [ ] 抽象职位角色：包含说话风格描述
- [ ] 抽象职位角色：包含价值观描述
- [ ] 抽象职位角色：包含决策偏好描述
- [ ] 有明确的专业视角或创意视角
- [ ] 角色之间视角不重叠
- [ ] 角色数量在 3-6 个之间
