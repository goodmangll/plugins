# 调研官机制

## 调研官角色

**必须包含**调研官(Research Officer)作为参与角色。

**调研官的职责**：
1. 在会议开始前收集材料和背景信息
2. 基于收集的材料分析"这个世界上哪些人最懂这个问题"，为角色选择提供依据
3. 在会议中验证其他参会人提供的证据是否正确
4. 补充缺失信息

## 工作流程

### 材料收集阶段(会议开始前)

调研官在角色决定阶段前执行材料收集：

1. **识别信息需求**
   - 议题涉及的技术领域
   - 需要了解的配置格式
   - 需要参考的文档

2. **信息收集**
   - 查找本地文档(优先)
   - 查找现有实现示例
   - 查找官方规范

3. **角色建议分析**
   - 基于收集的材料思考："这个世界上哪些人最懂这个问题？"
   - 为角色决定阶段提供建议

### 验证阶段(会议中)

调研官在以下关键阶段后执行验证：

| 阶段 | 调研官行动 |
|------|----------|
| 角色发言后 | 验证事实陈述，标注错误，请求补充 |
| 观点交换后 | 验证回应内容，检查冲突 |
| 共识提炼后 | 验证共识陈述的准确性 |

## 如何启用

在角色决定阶段，将调研官加入参与者列表：

```
参与者角色：
- [角色1]
- [角色2]
- [角色3]
- 调研官(必须)
```

## 工作流程集成

调研官的审核结果会影响讨论流程：

- 发现错误 → 暂停讨论，请求纠正
- 发现缺失信息 → 暂停讨论，触发调研补充
- 发现冲突 → 标记矛盾，请求澄清

## 材料收集保障

在角色决定阶段前，确保：

- 议题相关的技术规范已收集
- 现有实现示例已找到
- 必要的文档已读取
- 信息已整理并分发给各角色
- 角色建议已基于收集的材料形成

只有以上都完成，才能进入角色决定阶段。
