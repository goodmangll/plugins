# 角色决定流程（通用）

## 核心问题

协调器需要回答：

> **"这个世界上什么人适合探讨这个议题？"**

---

## 决定流程

### 第一步：议题分析

#### 1.1 理解议题本质

分析议题的核心要素：
- 议题的核心问题是什么？
- 涉及哪些领域或维度？
- 需要什么类型的分析或创意？

#### 1.2 识别关键维度

识别议题涉及的关键维度：
- 技术维度：架构、实现、性能、安全
- 商业维度：市场、成本、收益、竞争
- 用户体验维度：交互、视觉、流程、满意度
- 风险评估维度：安全风险、技术风险、商业风险
- 其他相关维度：法律、合规、伦理等

#### 1.3 分析输出

输出议题分析报告：

```
## 议题分析

**核心问题**：[议题的核心问题]

**涉及维度**：
- [维度1]：[说明]
- [维度2]：[说明]

**所需分析类型**：[圆桌会议分析/头脑风暴创意]
```

---

### 第二步：角色选择

根据议题分析结果，选择 3-6 个角色。

#### 2.1 选择原则

1. **多样性原则**：角色应该代表不同的视角和立场
2. **覆盖性原则**：覆盖议题的关键维度
3. **互补性原则**：角色之间有互补性
4. **避免重复**：避免角色视角过于重叠

#### 2.2 角色-维度映射

为每个关键维度分配角色，确保覆盖性：

| 维度 | 分配角色 | 角色类型 |
|------|----------|----------|
| [维度1] | [角色1] | [具体人物/抽象职位] |
| [维度2] | [角色2] | [具体人物/抽象职位] |

---

### 第三步：角色描述

#### 3.1 具体人物角色描述

对于具体人物角色，提供以下描述：

```
角色：[真实人物姓名]

身份描述：
- 职业/身份
- 主要成就或领域
- 代表性作品或项目

专业视角/创意视角：
- 核心思维模式
- 关注的重点领域
- 独特的观点或方法

说话风格：
- 语言特点
- 表达方式
- 情绪特征
```

#### 3.2 抽象职位角色描述

对于抽象职位角色，必须提供完整的人物画像：

```
角色：[具体化的职位名称]

身份描述：
- 专业背景和经验
- 专注领域
- 代表性能力

专业视角/创意视角：
- 核心分析角度
- 关注的重点维度
- 独特的见解

性格特征：
- [性格特征1]
- [性格特征2]

说话风格：
- [说话风格1]
- [说话风格2]

价值观：
- [价值观1]
- [价值观2]

决策偏好：
- [决策偏好1]
- [决策偏好2]
```

#### 3.3 角色表格输出

输出角色描述表格：

| 角色 | 身份描述 | 专业视角/创意视角 | 性格特征 | 说话风格 |
|------|----------|-------------------|----------|----------|
| [角色1] | [描述] | [视角] | [特征] | [风格] |
| [角色2] | [描述] | [视角] | [特征] | [风格] |

## 流程最佳实践

### 流程执行技巧

1. **分析全面性**：确保议题分析覆盖所有相关维度
2. **角色多样性**：角色组合应覆盖议题的关键维度
3. **角色互补性**：角色之间应有不同的视角和立场
4. **避免重复**：确保角色视角不重叠

### 常见错误

- ❌ 议题分析不全面，遗漏关键维度
- ❌ 角色之间视角过于重叠
- ❌ 角色数量过少（<3）或过多（>6）
- ❌ 角色与议题相关性低

### 质量检查

在确定角色后，进行以下检查：

- [ ] 角色数量在 3-6 个之间
- [ ] 角色代表不同的视角
- [ ] 覆盖议题的关键维度
- [ ] 角色之间有互补性
- [ ] 具体人物角色：有真实姓名和已知个性
- [ ] 抽象职位角色：包含完整的人物画像
- [ ] 角色与议题高度相关

## 输出要求

### 最终输出格式

```
## 角色决定结果

### 议题分析

[议题分析报告]

### 选择的角色

[角色描述表格]

### 角色选择说明

[说明选择这些角色的原因]
```

### 质量标准

1. **数量**：3-6 个角色
2. **多样性**：角色应该代表不同的视角
3. **描述清晰**：每个角色都需要明确的身份和视角描述
4. **相关性**：角色选择必须与议题/问题高度相关
5. **完整性**：抽象角色必须包含完整的人物画像
